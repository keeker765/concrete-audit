# 特征匹配方法对比报告

## 1. 测试概述

本报告对比 6 种特征匹配方法在混凝土试块干湿匹配场景下的表现，测试维度包括 **旋转不变性**、**跨外观匹配能力**、**跨批次区分度**。

---

## 2. 方法原理

### 2.1 SuperPoint + LightGlue（当前使用）

**SuperPoint**（2018, DeTone et al.）是一种基于深度学习的关键点检测+描述符提取器。

- **原理**：用一个全卷积网络（FCN）同时输出关键点热力图和 256 维描述符。训练过程分两步：先用合成几何图形（三角、矩形等）做自监督预训练（MagicPoint），再通过 Homographic Adaptation 在真实图像上做数据增强自蒸馏，生成伪标签训练最终模型。
- **特点**：关键点可重复性高，描述符区分度强。但 **不具备旋转不变性**——训练时没有对旋转做数据增强，网络依赖像素的局部空间排列，旋转会打乱这个排列。
- **旋转敏感原因**：卷积核是方向固定的（水平/垂直边缘检测），旋转 90° 后同一纹理的响应模式完全不同，导致描述符不匹配。

**LightGlue**（2023, Lindenberger et al.）是 SuperGlue 的轻量级改进版。

- **原理**：Transformer-based 图匹配网络。输入两组关键点+描述符，通过自注意力（self-attention）和互注意力（cross-attention）交替计算，输出匹配对。核心是学习了一种 **上下文感知的匹配策略**——不仅看局部描述符相似度，还看全局空间一致性。
- **特点**：自适应推理深度（简单图像对早停），比 SuperGlue 快 2-5×。支持 SuperPoint 和 DISK 两种前端。

### 2.2 DISK + LightGlue

**DISK**（2020, Tyszkiewicz et al.）是另一种学习型关键点+描述符提取器。

- **原理**：用强化学习（policy gradient）训练关键点检测策略。核心想法是"让关键点检测器和匹配器联合优化"——检测器的奖励信号来自匹配是否正确。描述符是 128 维。
- **vs SuperPoint**：DISK 在纹理丰富的区域检测更密集的关键点，在纹理贫乏区域更少。
- **旋转不变性**：**不具备**，和 SuperPoint 相同原因。

### 2.3 ORB（Oriented FAST and Rotated BRIEF）

**ORB**（2011, Rublee et al.）是 OpenCV 内置的传统特征提取器。

- **关键点检测**：用 FAST（Features from Accelerated Segment Test）检测角点——看一个像素周围的圆弧上是否有连续 N 个像素都比中心亮/暗。用图像金字塔实现多尺度检测。
- **方向计算**：用 **质心法（Intensity Centroid）** 计算每个关键点的主方向——以关键点为中心计算局部区域的亮度质心，从关键点到质心的向量方向即为主方向。
- **描述符**：BRIEF（Binary Robust Independent Elementary Features）的旋转版本。BRIEF 是比较关键点周围若干像素对的亮度大小，输出 0/1 二值串。ORB 在计算 BRIEF 时先按主方向旋转采样模式，使得描述符**旋转不变**。
- **旋转不变性**：**✅ 完全旋转不变**——因为每个关键点计算了主方向，描述符相对于主方向计算。
- **匹配方式**：Hamming 距离（异或运算，极快）。
- **弱点**：二值描述符区分度不如浮点描述符，在外观变化大的场景（如干湿混凝土）容易产生大量误匹配。

### 2.4 SIFT（Scale-Invariant Feature Transform）

**SIFT**（2004, Lowe）是经典的特征提取方法。

- **关键点检测**：在高斯差分（DoG, Difference of Gaussians）金字塔中找极值点——即在不同尺度和空间位置上亮度变化最显著的点。这保证了 **尺度不变性**。
- **方向计算**：计算关键点邻域的梯度方向直方图（36 个 bin），峰值方向即为主方向。如果有多个峰值，同一位置会产生多个方向不同的关键点。
- **描述符**：以主方向为参考系，将关键点周围 16×16 区域分成 4×4 子区域，每个子区域计算 8 方向梯度直方图，拼成 4×4×8 = 128 维浮点向量。
- **旋转不变性**：**✅ 旋转不变**——描述符相对于主方向计算，旋转后主方向跟着转，描述符保持一致。
- **匹配方式**：L2 距离 + Lowe's ratio test（最近邻距离 / 次近邻距离 < 0.75）。
- **弱点**：计算慢（浮点运算 + 金字塔构建），关键点数量较少，在大外观变化场景下匹配数不足。

### 2.5 LoFTR（Detector-Free Local Feature Matching with Transformers）

**LoFTR**（2021, Sun et al.）是一种无检测器的稠密匹配方法。

- **原理**：不先检测关键点，而是直接在特征图上做稠密匹配。流程：
  1. **CNN 骨干网络**提取两张图的多尺度特征图（1/8 分辨率）
  2. **Transformer 模块**（自注意力 + 互注意力）在粗粒度特征图上建立全局对应关系
  3. **粗匹配**：在 1/8 分辨率特征图上找置信度最高的匹配对
  4. **精细化**：将粗匹配区域裁出，在原始分辨率上做亚像素级精细化
- **vs SuperPoint+LightGlue**：LoFTR 不需要关键点检测，可以在纹理贫乏的区域（如天空、白墙）也产生匹配，因为 Transformer 利用全局上下文推断对应关系。
- **旋转不变性**：**❌ 不具备**——CNN 骨干网络和位置编码都是方向敏感的，旋转后特征响应不同。
- **弱点**：计算量大（Transformer 对 1/8 特征图做全注意力），对输入分辨率敏感。

## 3. 旋转不变性测试

同一张干态图（日升集团#1），旋转 0°/90°/180°/270° 后与原图匹配：

| 方法 | 0° | 90° | 180° | 270° | 旋转不变 |
|------|-----|------|------|------|---------|
| SuperPoint+LG | 1172 | 20 | 23 | 25 | ❌ |
| DISK+LG | 2048 | 33 | 36 | 62 | ❌ |
| LoFTR | 1920 | 16* | 28* | 18* | ❌ |
| ORB | 1434 | 1434 | 1434 | 1434 | ✅ |
| SIFT | 465 | 427 | 418 | 430 | ✅ |

*LoFTR 的数字是 confidence > 0.5 的匹配数

**结论**：所有深度学习方法（SuperPoint、DISK、LoFTR）都不具备旋转不变性。只有传统方法（ORB、SIFT）通过显式的主方向计算实现了旋转不变。

## 6. 结论

1. **SuperPoint+LightGlue 仍然是当前最佳选择**：虽然召回率只有 56%，但跨批次零误判，分数分布有清晰间隔
2. **RoMa 召回率最高（89%）但不可直接使用**：DINOv2 的语义特征导致所有混凝土试块看起来相似，跨批次分数 0.53-0.74 与同批分数 0.44-1.00 严重重叠
3. **ALIKED+LG 和 LoFTR 均不如 SP+LG**：ALIKED 在此场景产生的匹配点太少（~78 vs SP 的几百），LoFTR 分数系统性偏低
4. **MatchAnything 表现最差（6% 召回率）**：虽然原始匹配数很多（数百至数千），但置信度系统性偏低（mean ~0.3-0.5），RANSAC 内点率也低。跨模态预训练对干湿混凝土场景无明显帮助，640px 分辨率限制（8GB VRAM）可能也是因素
5. **所有深度学习方法均不具备旋转不变性**，需要4候选旋转搜索策略

### 可能的改进方向
- 提高 SP+LG 召回率：多尺度特征提取、增加 max_keypoints、调整 filter_threshold
- 融合策略：SP+LG 作为主判定 + RoMa 高阈值（>0.85）辅助
- 长期方向：收集足够干湿配对数据 → 训练 Siamese Network（见第 7 节）

## 7. 已测试但未采用的方法 & 未来方向

### 7.1 MatchAnything（浙大, 2025）— 已测试，效果不佳

**论文**: He et al., "MatchAnything: Universal Cross-Modality Image Matching with Large-Scale Pre-Training", 浙江大学 CAD&CG 实验室

**技术架构**：
- Backbone: RepVGG + EfficientLoFTR（LoFTR 的高效版本，使用局部注意力替代全局注意力）
- 训练策略: 大规模跨模态图像对预训练 → 通用跨模态特征表示
- 输出: 稠密匹配点对 + 置信度
- 模型大小: eloftr 权重 61MB，推理速度 ~0.5s/对

**测试配置**：
- 接入完整预处理管线（ROI 裁切 + 透视矫正 + 旋转对齐 + 贴纸区域过滤）
- 输入分辨率限制 640px（8GB VRAM 限制，更高分辨率会 OOM）
- 置信度过滤阈值 0.2，coarse match threshold 0.1
- 脚本: `test_matchanything.py`

**测试结果**：1/18 SAME（6% 召回率）—— 所有方法中最差

**失败分析**：
1. **置信度系统性偏低**：mean_conf 普遍在 0.3~0.5，远低于 SP+LG 的 0.5~0.6
2. **分辨率受限**：640px 可能丢失关键纹理细节，但提高到 832+ 会爆显存
3. **跨模态预训练域不匹配**：MatchAnything 训练于红外/可见光/SAR 等真正跨模态场景，干湿混凝土的纹理差异属于同模态外观变化，不在其优势领域
4. **局部注意力限制**：EfficientLoFTR 使用局部注意力替代全局注意力，在小 ROI 图像上可能丢失全局对应关系

**结论**：不建议集成到主管线。尽管理论上跨模态预训练应有优势，但实测表现远不如 SP+LG。

### 7.2 DeDoDe v2

**论文**: Edstedt et al., "DeDoDe v2: Analyzing and Improving the DeDoDe Detector and Descriptor", 2024

**技术特点**：
- 关键点检测器 + 描述符分离设计（与 SuperPoint 类似但架构不同）
- 使用 DualSoftMax 匹配策略替代传统最近邻匹配
- 在 MegaDepth benchmark 上性能优于 SuperPoint

**未测试原因**：
- kornia 集成的 DeDoDe API 不包含内置 matcher（只有 `detect()` + `describe()`，需额外集成 `DualSoftMaxMatcher`）
- 独立包安装与 kornia 版本冲突
- 设计目标是自然场景 pose estimation，对混凝土低纹理表面的适用性存疑

### 7.3 学术研究：表面纹理指纹识别

我们的问题本质上是 **"物体再识别（Object Re-identification）via 表面纹理指纹"**，学术界已有类似场景的研究：

#### 最相关的已有工作

| 论文 | 场景 | 方法 | 与我们的关联 |
|------|------|------|-------------|
| Rutinowski et al., 2021 (IEEE ETFA) | 木质托盘表面纹理 → 仓库物体再识别 | 控制光照拍照 → 纹理 fingerprint → Re-ID | **最接近**：木纹唯一性 ≈ 骨料分布唯一性 |
| Robert, 2020 | 树皮纹理 → 个体树木再识别 | 深度学习特征描述符 | 自然纹理 Re-ID |
| Filip et al., 2024 (arXiv:2410.13615) | 材料表面 → 感知特征编码 | Material Fingerprinting 编码器 | 材料级特征表示 |
| Shim et al., 2023 | 3D打印表面纹理 → 打印机源识别 | 层状纹理编码器 (Layered Texture Encoder) | 制造过程留下的唯一"指纹" |

#### 对我们的启发

**当前方法（Feature Matching）的局限**：SuperPoint+LG 等方法设计目标是场景级 pose estimation（如 SLAM、SfM），并非为物体再识别优化。它们关注"几何一致性"而非"纹理独特性"。

**更契合的技术路线**（如果未来需要更高准确率）：

| 方向 | 方法 | 优势 | 需要条件 |
|------|------|------|---------|
| **Siamese Network** | 孪生网络 + 对比损失/三元组损失 | 直接学习"同一块 vs 不同块"的判别特征 | 需要标注的干湿配对数据（正负样本） |
| **Texture Encoding** | LBP / GLCM 等纹理统计特征 | 传统方法，可解释性强，可作为辅助特征 | 无需训练 |
| **Re-ID Embedding** | 类似行人 Re-ID 的 embedding 网络 | 端到端学习跨条件不变的表面特征 | 需要大量配对数据 |

**推荐路线**：
1. **短期**：继续用 SuperPoint+LG + 调参优化（当前最优）
2. **中期**：收集 50+ 组干湿配对数据 → 训练 Siamese Network
3. **长期**：开发专用的"混凝土表面指纹"编码器（可发论文）

## 8. 评分指标计算原理

### 8.1 从特征点到最终分数的完整流程

```
原始图像对
   ↓ [特征提取] SuperPoint / ALIKED / SIFT 等
关键点集合 kpts₀, kpts₁（各 ~2048 个点）+ 描述符
   ↓ [LightGlue 匹配]
原始匹配对：{(i, j)} ，附带置信度 sᵢ ∈ [0,1]  → N_raw 对
   ↓ [贴纸区域过滤]
移除两端点都落在蓝色贴纸 mask 内的匹配
→ N_filtered 对（过滤后有效匹配数）
   ↓ [RANSAC 单应性估计]
cv2.findHomography(pts₀, pts₁, RANSAC, reprojError=5.0)
→ 内点 mask（每对匹配标记为 inlier=1 / outlier=0）
→ N_inliers 个几何一致的内点
   ↓ [综合评分]
final_score
```

### 8.2 评分公式

**两个子指标**：

| 指标 | 计算方式 | 含义 |
|------|---------|------|
| `mean_conf` | 所有匹配对置信度的均值：`mean(sᵢ)` | LightGlue 认为这批匹配有多可信 |
| `inlier_ratio` | RANSAC 内点比率：`N_inliers / N_filtered` | 匹配在几何上有多一致（单应性误差 < 5px） |

**合并为最终分数**：

```
raw_score = 0.5 × mean_conf + 0.5 × inlier_ratio

if N_filtered < MIN_MATCHES (15):
    final_score = raw_score × (N_filtered / 15)   # 匹配数不足时线性惩罚
else:
    final_score = raw_score
```

**判决阈值**：

| final_score | N_filtered | 判决 |
|-------------|------------|------|
| ≥ 0.60 | ≥ 4 | **SAME**（同一试块） |
| < 0.60 | ≥ 4 | **DIFFERENT**（不同试块） |
| — | < 4 | **INSUFFICIENT**（匹配点太少，无法判断） |
| — | — | **INVALID**（贴纸/QR检测失败，未进行匹配） |

### 8.3 举例说明

**典型 SAME 案例**（象山县美#1，SP+LG）：
```
N_raw = 512 → 过滤贴纸 → N_filtered = 487
RANSAC: N_inliers = 403, inlier_ratio = 403/487 = 0.827
mean_conf = 0.873
raw_score = 0.5 × 0.873 + 0.5 × 0.827 = 0.850
N_filtered(487) ≥ 15 → final_score = 0.850 ✅ SAME
```

**典型 DIFFERENT 案例**（宁波大目#2，SP+LG）：
```
N_raw = 89 → N_filtered = 76
RANSAC: N_inliers = 23, inlier_ratio = 23/76 = 0.303
mean_conf = 0.365
raw_score = 0.5 × 0.365 + 0.5 × 0.303 = 0.334
final_score = 0.334 ❌ DIFFERENT
```

### 8.4 为什么 RANSAC 内点率是关键

RANSAC（随机采样一致性）估计单应矩阵 H（3×3，8自由度）：

- **同一试块**：干湿图像的纹理对应关系满足全局刚性变换（加上轻微形变），H 能拟合大多数匹配，内点率高（>0.7）
- **不同试块**：匹配对是随机错误的，不存在一个 H 能同时满足大量匹配，内点率低（<0.4）

**mean_conf 单独不够可靠**：LightGlue 在不同批次的混凝土图片上仍可能给出 0.5-0.6 的置信度（因为表面纹理相似），但 RANSAC 会揭穿这些匹配在几何上不一致。两者结合后分辨率更高。

## 6. 为什么深度学习方法不旋转不变？

核心原因是 **卷积神经网络的平移等变性不等于旋转等变性**：

1. **卷积核是方向固定的**：一个检测"水平边缘"的 3×3 滤波器，旋转 90° 后变成检测"垂直边缘"。同一个纹理旋转后，在特征图上的响应模式完全不同。

2. **位置编码是方向敏感的**：LightGlue 和 LoFTR 都使用位置编码告诉 Transformer "这个特征在图像的什么位置"。旋转后位置关系改变，注意力计算结果不同。

3. **训练数据的分布**：这些模型在训练时看到的图像对大多是小角度变化（自然场景中相机运动通常不会产生 90° 旋转），所以模型没有学到处理大角度旋转的能力。

传统方法（SIFT、ORB）通过 **显式计算每个关键点的主方向** 并相对于主方向提取描述符来实现旋转不变。这是一种工程上的显式处理，而不是让网络自动学习。
